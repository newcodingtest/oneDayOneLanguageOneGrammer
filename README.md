## 🚀 Why This Project (왜 만들었는가)

이 프로젝트는 단순한 영어 학습 기능 구현이 아니라,  
"사용자가 실제로 사용하는 서비스는 어떻게 만들어지는가?”를 검증하기 위해 시작했습니다.

기존에는 백엔드 개발자로서 **기능 구현과 성능 개선**에 집중해왔지만,  
이 프로젝트를 통해 아래와 같은 질문을 스스로 던지게 되었습니다:

- 사용자는 어떤 경로로 유입되는가?
- 학습 서비스는 어떻게 반복 사용을 유도할 수 있는가?
- 이 서비스는 어떻게 수익화할 수 있는가?

👉 **“개발 → 사용자 → 데이터 → 개선”의 전체 사이클을 경험하는 것이 목적**이었습니다.

## 💡 얻은 인사이트 (Key Insights)

### 1. 기능보다 “지속성”이 중요하다

- 단순 문법 제공보다  
  👉 “매일 1개” 구조가 **재방문을 유도**

### 2. 진입 장벽이 낮아야 한다

- 로그인 없음
- 즉시 사용 가능  
  → 초기 사용자 확보에 매우 중요

### 3. 콘텐츠는 자동화되어야 스케일된다

- AI 기반 자동 생성  
  → 운영 비용 최소화 + 지속적인 콘텐츠 공급 가능

### 4. 개발자의 관점 변화

이전:

> 기능을 잘 만드는 것이 중요

이후:

> 사용자가 **왜 들어오고, 왜 나가는지**가 더 중요

## 🧠 기술 선택 이유 (Why Next.js)

Java / Spring 기반 백엔드 개발자임에도 불구하고  
이 프로젝트에서는 **Next.js 스택을 선택**했습니다.

### 1. Zero Infrastructure

- 별도 서버 구축 불필요
- 배포 = Git push (Vercel)
- 운영 부담 최소화

### 2. Zero Backend (Minimal Backend)

- API Route로 서버 역할 수행
- DB 없이도 캐싱 + AI로 서비스 구성 가능

### 3. 빠른 MVP 검증

- 아이디어 → 배포까지 빠른 사이클
- 실험 비용 최소화

👉 **사용자 반응을 빠르게 검증하기 위한 스택 선택**
